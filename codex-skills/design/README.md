## GLOBAL RULES

- Do not make large unnecessary changes
- Edit only relevant files
- Do not delete code unless asked
- Do not change configs or dependencies

### Code

- Follow existing patterns
- Avoid unnecessary abstractions
- Prefer simple solutions

### UI
- Clean, non-generic design
- Responsive always
- Clear hierarchy
- No overdesign

### Behavior
- If unsure → safe approach
- Keep working logic intact

### Output
